# Writing

Short > long. Specific > general. Active > passive.

## Tickets
Lead with the change. AC must be testable (if you can't write a test, rewrite it).
Include: what to do (1-2 sentences), testable AC, blocked-by if relevant.
Exclude: "This ticket involves..." / universal criteria / PRD restatements.

## Commit Messages
Subject: imperative mood, <=72 chars, no period. Body: only when why isn't obvious. Never "This commit...".
- "Fix login redirect after password reset" YES
- "Fixed the login redirect issue" NO

## Docs
One sentence > one paragraph. Write what reader needs, then stop.
