# Writing Rules (for rules authors)

## Structure
```
# Name
## Principle - core idea 1-4 sentences
## Rules/Patterns - tables, checklists, decision trees
## Checklist - pre-action verification
```

## Tone
Prescriptive not suggestive. "Do X" not "Consider X". "Never Y" not "generally avoid Y".
Token-efficient: delete unnecessary words, no fluff, no repetition.

## Length
Target 30-80 lines. >150 lines -> split or trim.

## When to create
Create: distinct principle, enforceable, broadly applicable, actionable.
Don't: detail of existing rule, too subjective, project-specific, one-time decision.
